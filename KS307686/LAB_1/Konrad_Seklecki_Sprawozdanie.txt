Zadania do wykonania
Zainstaluj klienta Git i obsługę kluczy SSH
>which git //SS1
Sklonuj repozytorium https://github.com/InzynieriaOprogramowaniaAGH/MDO2022 za pomocą HTTPS
git clone https://github.com/InzynieriaOprogramowaniaAGH/MDO2022.git //SS2
Upewnij się w kwestii dostępu do repozytorium jako uczestnik i sklonuj je za pomocą utworzonego klucza SSH
Utwórz dwa klucze SSH, inne niż RSA, w tym co najmniej jeden zabezpieczony hasłem
ssh-keygen -t ed225519 -c "galanonim856@gmail.com"
kay_one
*password
*password //SS3 SS4
Skonfiguruj klucz SSH jako metodę dostępu
ssh-add ~/.ssh/id_ed25519 //SS5
<dodanie klucz do konta github> //SS6
Sklonuj repozytorium z wykorzystaniem protokołu SSH
git clone git@github.com:InzynieriaOprogramowaniaAGH/MDO2022.git //SS7
Przełącz się na gałąź swojej grupy
git checkout MDO2022
git checkout GCL03 //SS8
Utwórz gałąź o nazwie "inicjały & nr indeksu" np. KD232144
git branch KS307686
git checkout KS307686 //SS9 SS10
Rozpocznij pracę na nowej gałęzi
W katalogu właściwym dla grupy utwórz nowy katalog, także o nazwie "inicjały & nr indeksu"np. KD232144
mkdir KS307686
git add . //SS11
W nowym katalogu dodaj plik ze sprawozdaniem
vim Konrad_Seklecki_Sprawozdanie.txt
git add Konrad_Seklecki_Sprawozdanie.txt //SS12
Dodaj zrzuty ekranu
git add . 
git status //SS13
Wyślij zmiany do zdalnego źródła
git commit -m 'Ready'//SS14
Spróbuj wciągnąć swoją gałąź do gałęzi grupowej

Zaktualizuj sprawozdanie i zrzuty o ten krok i wyślij aktualizację do zdalnego źródła (na swojej gałęzi)
git add Konrad_Seklecki_Sprawozdanie.txt
git add SS14.png
git commit --amend
Oznacz tagiem ostatni commit i wypchnij go na zdalną gałąź
git tag end_is_near
